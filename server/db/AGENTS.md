# server/db — Database Schema and Migrations

## Nav
| file/dir | role |
|----------|------|
| `schema.json` | Single source of truth for all DB table definitions |
| `event_log_definitions.json` | TrId master document — param semantics per event type |
| `migrations/` | SQL files generated by gen:orm (timestamped, review before prod deploy) |

## Rules
FILE: `schema.json` (single file for all tables in this DB)
CMD:  `npm run gen:orm` — validates schema — generates migration SQL — optionally executes

- Edit `schema.json` to add/modify tables — run `npm run gen:orm`
- gen-orm ADDS new tables and columns — never auto-drops (data safety)
- Column type changes are warned but NOT auto-executed — alter manually
- `dry_run=true` (default): SQL file generated only, DB not touched
- `dry_run=false`: SQL generated AND executed on connected DB
- No changes detected: gen-orm skips SQL file creation (no empty migration files)

## schema.json Format
```json
{
  "database": "projectflood_db",
  "output": {
    "namespace": "ProjectFlood.Infrastructure.Generated",
    "context": "AppDbContext"
  },
  "tables": [
    {
      "name": "table_name",
      "comment": "optional description",
      "conflict": "ignore",
      "columns": [
        { "name": "id", "type": "int64", "pk": true, "auto": true, "null": false }
      ],
      "indexes": [
        { "name": "idx_name", "columns": ["col1"], "unique": false }
      ]
    }
  ]
}
```

## Column Types
int8/16/32/64 | uint8/16/32/64 | float | double | bool
string | string(N) | datetime | date | json | bytes

## Conflict Resolution
| value | generated method | use when |
|-------|-----------------|----------|
| `"ignore"` | `InsertIgnoreAsync(row, ct)` | concurrent insert of same unique key must silently no-op |

### Middleware-Level Concurrent Insert Risk
Any DB `Insert` inside middleware (before `UserSerializeFilter`) is exposed to concurrent-insert races.
**Fix**: Add `"conflict": "ignore"` to the table in `schema.json` → gen:orm generates `InsertIgnoreAsync`.

**Tables with `conflict: ignore`** (update as new tables are added):
| table | unique key | insert site |
|-------|-----------|-------------|
| `sessions` | `session_id` | `SessionValidationMiddleware` |
| `players` | `user_id` (PK), `platform_pid` | `UserIdResolutionMiddleware` |
